identities sealed

The single-demo strategy: real Makro becomes the one demo shown to every prospect — with identities masked. Real structure, real tenure bands, real cycle; no name, no phone, no NRC. For Makro's own HRBP walkthrough the same views run unmasked: their data, their people. The mask is only the third-party layer.

What this page is — and is not

This is the masking spec rendered on placeholders (··, ——, •). It is not a person record and contains no identity — real or invented. It defines how the real portal looks to a prospect, so the same live system is safe to show a competitor.

MODE 1 · FOR MAKRO
Unmasked — their own data

Full names, full records, the client-visible Change Log. Shown only in the HRBP's own walkthrough of their own tenant — never on a forwardable frame.

MODE 2 · FOR PROSPECTS
Masked — the same system, sealed

Initials-only identity, role labels, hidden phone and NRC. Real structure, tenure bands and cycle underneath — the proof stays powerful because it stays true.
